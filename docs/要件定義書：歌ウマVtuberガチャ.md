# **プロジェクト名：歌ウマVtuberガチャ**

歌が上手いVtuberとの「一期一会」を演出する、24時間更新のガチャサイト。

## **1\. サービス概要**

* **ターゲット**: 新しい歌ウマVtuberを探しているリスナー  
* **核心的価値**: AIによるパーソナライズされた紹介と、ユーザー全員でガチャ時間を早める協力体験

## **2\. 機能要件**

### **A. メイン機能（ガチャ）**

* **24時間周期の排出**: サーバー時刻に基づき、24時間に1回新しいVtuberを決定。  
* **YouTube Data API連携**: 事前にリストアップした、または条件に合うVtuberの動画情報を取得。  
* **Gemini AI紹介文生成**:  
  * 取得した動画タイトルや概要欄、チャンネル情報をGemini APIに渡し、そのVtuberの魅力を伝える紹介文を自動生成。  
* **アーカイブ保存**: 排出されたVtuber情報（名前、動画URL、紹介文、排出日）をSupabaseへ自動保存。

### **B. サブ機能（ソーシャル・回遊）**

* **応援ハートボタン**:  
  * 全ユーザーがクリック可能。  
  * クリックの総数が一定数に達するごとに、次回のガチャ排出までのカウントダウンを短縮（例：1000クリックで10分短縮）。  
* **アーカイブ検索**: 過去に排出されたVtuberを一覧表示し、キーワードや日付で検索。  
* **シェア機能**: ガチャ結果をX（旧Twitter）などでシェア。

## **3\. 技術スタック**

* **Frontend**: React または Next.js (Cloudflare Pagesでホスト)  
* **Backend/Database**:  
  * **Supabase**:  
    * データベース（アーカイブ、クリック統計、現在のガチャ状態）  
    * Edge Functions（24時間タイマーの制御、Gemini/YouTube APIの呼び出し）  
* **AI**: Google AI Studio (Gemini 1.5 Flash等)  
* **External API**: YouTube Data API v3  
* **Infrastructure**: Cloudflare Pages / GitHub

## **4\. データ構造（Supabase案）**

* vtubers: Vtuberの基本マスターデータ（手動またはスクレイピングで蓄積）  
* gacha\_history: 過去に排出された結果（紹介文含む）  
* global\_stats: 現在のカウントダウン、総クリック数、次回排出予定時刻

## **5\. 開発ステップ（フェーズ1: MVP）**

1. **DB構築**: Supabaseでテーブル作成。  
2. **API連携テスト**: YouTube APIで動画取得 → Geminiで紹介文生成のフローをスクリプトで確認。  
3. **ガチャロジック実装**: Supabase Edge Functionsで「24時間ごとに1件選出」する機能を実装。  
4. **UI実装**: ガチャ画面（カウントダウン、Vtuber表示）とアーカイブ画面の作成。  
5. **クリック短縮実装**: リアルタイムでクリック数を反映し、タイマーを動的に計算。
